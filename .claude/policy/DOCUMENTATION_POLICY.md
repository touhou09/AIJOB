# 문서화 정책

> 모든 ingkle 저장소에 적용되는 문서화 기준

## 문서 분류

| 위치 | 목적 | 대상 독자 | 예시 |
|------|------|----------|------|
| `.claude/work/` | 작업 재현 | Claude (AI) | 변경 파일, 명령어, 미완료 사항 |
| `docs/` | 지식 공유 | 사람 + Confluence | 가이드, 런북, 아키텍처 설명 |
| `.claude/DECISIONS.md` | 결정 이력 | Claude + 사람 | 왜 이렇게 결정했는지 |

## .claude/work/ 포맷

```markdown
## {날짜} | {작업 제목}
- 변경: {파일 경로 및 변경 내용}
- 패턴: {사용한 명령어/방법}
- 미완료: {남은 작업과 이유}
- 이슈: {발견한 문제}
```

## docs/ 포맷

```markdown
# {제목}

## 배경
{왜 이 문서가 필요한지}

## 방법
{구체적인 절차}

## 주의사항
{실수하기 쉬운 부분, 알아야 할 제약}
```

## .claude/DECISIONS.md 포맷 (AD 번호 부여)

```markdown
## AD-{번호}: {결정 제목}

- **일시**: {날짜}
- **결정**: {무엇을 결정했는지}
- **이유**: {왜 이렇게 결정했는지}
- **상태**: 계획/진행 중/완료/폐기
- **참조**: {관련 파일/문서}
```

## 새 아키텍처 결정 발생 시

1. `.claude/DECISIONS.md`에 AD 번호로 추가
2. 필요 시 `docs/`에 상세 문서 작성
3. 관련 `work/{category}.md`에서 AD 번호 참조
