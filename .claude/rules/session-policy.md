# 세션 정책

## 세션 시작 절차 (Hot Tier 전부 로드)
1. `CLAUDE.md` 자동 로드 (Claude Code 기본 동작)
2. `.claude/STATE.md` 읽고 현재 상태 파악
3. `.claude/CONTEXT.md` 읽고 프로젝트 컨텍스트 파악
4. `.claude/DECISIONS.md` 읽고 결정 이력 파악
5. `.claude/TODO.md` 읽고 오늘의 작업 확인 (Jira Ready와 항상 일치)
6. 사용자 요청에 해당하는 `.claude/work/{category}.md` 로드

## 세션 종료 판단 흐름
```
작업 완료
  |
  +- 코드에서 유추 가능? -> 기록 안 함
  |
  +- 기록 대상
       |
       +- 이전과 동일 작업 + 새 이슈 없음 -> STATE.md만 갱신
       +- 이전과 동일 작업 + 새 트러블슈팅 -> work/troubleshooting.md + docs/
       +- 새 작업 계열
            +- 해당 카테고리 파일 존재 -> append
            +- 미존재 -> 새 카테고리 파일 생성 + CLAUDE.md 인덱스 갱신
```

## 기록 기준

### 기록하는 것
- 변경한 파일과 변경 내용 (재현 가능한 수준)
- 사용한 명령어/패턴
- 미완료 사항과 이유
- 발견한 이슈/주의사항

### 기록하지 않는 것
- 코드를 읽으면 알 수 있는 내용
- git log/blame으로 확인 가능한 내용
- 이전 기록과 중복되는 내용

## 세션 종료 자동 제안

Claude는 다음 조건을 감지하면 `/project:done` 실행을 자발적으로 제안한다:
- 사용자가 "끝", "완료", "다했다", "오늘은 여기까지", "마무리" 등 종료 시그널을 보낼 때
- 파일 변경 5개 이상 발생 후 추가 요청이 없을 때
- 커밋 완료 후 다음 지시가 없을 때

제안 형태: "작업이 완료된 것 같습니다. `/project:done`으로 문서화할까요?"
- 사용자가 거부하면 즉시 중단
- 짧은 Q&A 세션(파일 변경 없음 + 3턴 이하)에서는 절대 제안하지 않음
