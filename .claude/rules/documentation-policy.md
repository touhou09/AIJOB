# 문서화 정책

## 문서 분류

| 위치 | 목적 | 대상 독자 | 예시 |
|------|------|----------|------|
| `.claude/work/` | 작업 재현 | AI (Claude) | 변경 파일, 명령어, 미완료 사항 |
| `docs/` | 지식 공유 | 사람 + Confluence | 가이드, 런북, 아키텍처 설명 |
| `DECISIONS.md` | 결정 이력 | AI + 사람 | 왜 이렇게 결정했는지 |

## work/ 포맷
```markdown
## {날짜} | {작업 제목}
- 변경: {파일 경로 및 변경 내용}
- 패턴: {사용한 명령어/방법}
- 미완료: {남은 작업과 이유}
- 이슈: {발견한 문제}
```

## DECISIONS.md 포맷 (AD 번호 부여)
```markdown
## AD-{번호}: {결정 제목}
- **일시**: {날짜}
- **결정**: {무엇을 결정했는지}
- **이유**: {왜 이렇게 결정했는지}
- **상태**: 계획/진행 중/완료/폐기
- **참조**: {관련 파일/문서}
```

## work/ 생명주기 관리 (200줄 한도)
1. 해당 `work/{category}.md`의 총 줄 수 확인
2. 150줄 초과 시 정리 실행:
   - 완료 + 코드/git에서 유추 가능 -> 삭제
   - 결정 사항 -> DECISIONS.md로 이동 후 삭제
   - 미완료/진행 중 -> 유지
3. 200줄 초과 시 -> 오래된 완료 항목부터 강제 삭제

## 삭제하면 안 되는 것
- 미완료 작업 (TODO)
- 아직 유효한 주의사항/이슈
- DECISIONS.md에 없는 결정 사항 (옮긴 후 삭제)
